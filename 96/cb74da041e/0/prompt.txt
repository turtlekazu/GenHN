# Plan: Split gardener.js into themes file and app logic file

## Context

`gardener.js` は現在 2,315 行あり、テーマ定義（CSS文字列）とアプリロジックが混在している。
テーマを追加・編集したいユーザーが目的のコードに辿り着きにくいため、分離する。

## Split Strategy

| ファイル | 内容 | 行数（目安） |
|---------|------|------------|
| `presets.js` | `STYLE_PRESETS` オブジェクトのみ（14テーマのCSS定義） | ~800行 |
| `gardener.js` | その他すべて（SYSTEM_STYLE, PANEL_STYLE, AI prompts, HNData, App, init） | ~1,500行 |

`SYSTEM_STYLE` / `PANEL_STYLE` はユーザーが触らない内部フレームワークCSSなので `gardener.js` に残す。
`STYLE_PRESETS` はテーマ追加・編集の対象なので `presets.js` に分離する。

## 実装手順

### 1. `presets.js` を新規作成
`gardener.js` の lines 703–1500 の `STYLE_PRESETS` 定義をそのまま切り出す：
```js
const STYLE_PRESETS = {
  "None": ``,
  "Minimalist": `...`,
  // ... 14テーマ
};
```

### 2. `gardener.js` から `STYLE_PRESETS` を削除
lines 703–1500 を削除し、代わりに `// STYLE_PRESETS is defined in presets.js` コメントを残す（任意）。

### 3. `index.html` のscriptタグを更新
```html
<!-- 現在 -->
<script src="gardener.js"></script>

<!-- 変更後（presets.js を先に読み込む） -->
<script src="presets.js"></script>
<script src="gardener.js"></script>
```
`presets.js` は `App` が参照するため、必ず `gardener.js` より先に読み込む。

## 依存関係の確認

`STYLE_PRESETS` を参照している箇所（すべて `App` 内）：
- App.init()  → `STYLE_PRESETS[savedPreset]`
- App.renderPresetButtons() → `Object.keys(STYLE_PRESETS)` / `STYLE_PRESETS[themeName]`
- App.applyStyle() → `STYLE_PRESETS[themeName]` / `for (const key in STYLE_PRESETS)`

グローバル変数として宣言するため、ファイル分割後もそのまま参照可能（変更不要）。

3. 各テーマのクリックでスタイルが適用されることを確認
4. localStorage に保存されたテーマが復元されることを確認
